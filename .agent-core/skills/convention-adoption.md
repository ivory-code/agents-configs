# Convention Adoption

레거시 코드베이스에 공통 컨벤션을 점진적으로 적용하는 스킬.

## 원칙

1. 신규 파일은 최신 컨벤션 100% 적용
2. 기존 파일은 수정 라인 중심 적용
3. 기능 변경과 대규모 리라이트를 분리

## 도입 단계

### Phase A: 안전한 표면부터

- import 순서
- 네이밍
- `any` 제거(수정 라인)

### Phase B: 구조 정리

- 스타일 분리
- query key factory 도입
- API/도메인 타입 명시화

### Phase C: 회귀 방지

- 핵심 플로우 테스트 추가
- PR 체크리스트 자동 적용

## PR 분할 규칙

- 기능 PR: 동작 변화 중심
- 정리 PR: 구조/스타일/타입 개선 중심
- hotfix PR: 최소 변경 + 후속 정리 티켓 필수

## 최소 이행 기준

- 신규/수정 라인 `any` 제거
- query key 하드코딩 제거
- 디자인 시스템 우선 사용
- profile 기반 검증 명령 통과

## 실패 패턴

- "이번만"으로 안티패턴 복제
- 기능 PR에 과도한 리팩터링 동시 포함
- 예외 사유/만료일 없이 규칙 무시
